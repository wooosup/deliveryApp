# 배달의 부족

---

## 프로젝트 개요

- 프로젝트 명 : 배달의 부족
- 이 프로젝트를 하는 이유
    - 어떤 도전을 해야 될지 고민하던 중 우테코에 들어가고 싶다는 나의 열망을 담아 우아한 형제의 배달의 민족을 모티브로 진행
- 목표 : 제출 마감 전까지 끊임없이 개발해서 Swagger로 API 문서화까지 완료하기
- 기술 스택
  - Java 17, Spring Boot 3.5.6, JPA, H2, Swagger

## 도메인

### Store

- [ ] 가게이름, 사장, 총 판매금액, 가게에 파는 메뉴들을 가진다.

### Product

- [ ] 가게, 메뉴이름, 가격을 가진다.
- [ ] 상품 판매상태, 상품 타입을 가진다. (enum 타입으로 구현)

### Owner

- [ ] 이름, 비밀번호를 가진다.
- [ ] 사장은 자신의 가게를 등록하고 관리한다.
- [ ] 사장은 상품을 등록/수정하고 판매상태를 변경한다.

### User

- [ ] 이름을 가진다.
- [ ] 주문을 생성할 수 있다.
- [ ] 주문 이력을 조회할 수 있다.

### Order

- [ ] 여러 상품을 한 번에 주문할 수 있다.
- [ ] 주문상품 목록과 결제 금액을 가진다.

### OrderProduct

- [ ] 주문과 상품의 다대다 관계를 해결하기 위한 중간 테이블
- [ ] 각 상품의 수량 및 금액을 가진다.

--- 

## 진행 목표 체크리스트

- [ ] 가게 CRUD, 총 판매금액 집계
- [ ] 상품 CRUD, 판매상태/타입 관리
- [ ] 손님 / 사장 등록 및 조회
- [ ] 주문 생성, 주문상품 처리, 결제 금액 계산
- [ ] Swagger로 API 문서화

--- 

## 실행 방법
- 빌드: ./gradlew build
- 실행: ./gradlew bootRun
- 테스트: ./gradlew test
